# Company QA Checklist

회사 PC에서 `npm` 없이 설치본으로 운영하기 위한 수동 점검 목록입니다.

## 0. 사전 조건

- 앱 설치본 확보 (`Worklog.app` 또는 배포된 설치 파일)
- 사내 Git Enterprise 계정/권한 확인
- 사내 인증 방식 준비 (SSH 키 또는 PAT)

## 1. 설치 확인

1. 앱이 정상 실행된다.
2. 첫 진입 시 저장소 설정 화면이 나온다.
3. 설정 화면에 경고 문구(사내 Git Enterprise 사용 권장)가 보인다.
4. 소스코드 다운로드 + `npm install` 경로를 사용하지 않았는지 확인한다.
5. 설치본 출처가 [latest-installer/LATEST.md](/Users/roycekim/royce_lab/worklog/latest-installer/LATEST.md)와 일치하는지 확인한다.

## 1-1. 웹앱 실행 확인

1. `npm run web` 실행이 성공한다.
2. 브라우저에서 `http://127.0.0.1:3210` 접속이 된다.
3. 초기 `bootstrap` 상태가 정상 응답한다.

## 2. 저장소 설정 확인

### 기존 로컬 저장소 모드

1. 저장소 경로 선택 가능
2. 브랜치 입력 후 설정 저장 성공
3. 상단 헤더에 `repoPath / branch / auto-push`가 표시됨
4. `허용 remote host 목록` 입력 후 반영되는지 확인
5. 중복 host 또는 형식 오류 host 입력 시 저장이 차단되는지 확인

### 원격 클론 모드

1. 사내 remote URL 입력 가능
2. 대상 폴더 선택 가능
3. 클론 + 브랜치 설정 성공
4. `허용 remote host 목록` 입력 가능
5. 클론 URL 입력 시 host가 자동 제안되는지 확인

## 3. 캘린더/에디터 기본 동작

1. 현재 월 캘린더가 표시된다.
2. 날짜 클릭 시 우측 편집 패널이 갱신된다.
3. 로그 없는 날짜는 기본 템플릿이 로드된다.
4. 기존 로그 날짜는 내용이 로드된다.
5. 에디터 하단의 Generated Markdown/Rendered Preview가 입력 내용과 동기화된다.
6. 월간 요약 카드가 해당 월 기준으로 갱신된다.

## 4. 저장 및 Git 동기화

1. 로그 내용 작성 후 저장 버튼 클릭
2. `logs/YYYY/MM/YYYY-MM-DD.md` 파일 생성/수정 확인
3. 커밋 메시지 `worklog: update YYYY-MM-DD` 확인
4. auto-push ON이면 사내 원격으로 push 성공
5. auto-push OFF이면 커밋까지만 수행

## 4-1. 검색 기능 점검

1. 상단 검색창에서 키워드 검색이 동작한다.
2. 날짜 문자열(`YYYY-MM-DD`) 검색이 동작한다.
3. 검색 결과 클릭 시 해당 날짜가 선택되고 에디터가 갱신된다.

## 4-2. PDF 내보내기 점검

1. `PDF 내보내기` 버튼 클릭 시 파일이 생성된다.
2. 생성 경로가 `exports/YYYY/MM/YYYY-MM-DD.pdf` 규칙을 따른다.
3. PDF 내용이 해당 날짜의 최신 편집 내용과 일치한다.

## 5. 오류 케이스 점검

1. 인증 오류 시 상태바에 인증 점검 안내가 나온다.
2. 브랜치 불일치 시 상태바에 브랜치 점검 안내가 나온다.
3. push 거절 시 원격 반영 필요 안내가 나온다.
4. merge/rebase 충돌 시 충돌 해결 안내가 나온다.

## 6. 외부 유출 방지 점검

1. `git remote -v` 결과가 사내 주소인지 확인
2. GitHub 주소가 remote로 설정되지 않았는지 확인
3. 앱 상태 검증 시 GitHub remote면 경고 메시지가 표시되는지 확인
4. GitHub remote일 경우 auto-push가 꺼지거나 push가 차단되는지 확인
5. 허용 host 외 remote일 경우 검증/푸시가 차단되는지 확인

## 7. 재설정 동작

1. 상단 `저장소 재설정` 버튼 동작 확인
2. 설정 변경 후 즉시 새 저장소에 대해 저장/커밋되는지 확인
3. 상단 `Preflight 검사` 버튼으로 상태가 갱신되는지 확인
4. 최근 Preflight 결과 5건 이력이 표시되는지 확인

## 7-1. 다중 프로필 동작

1. 헤더에서 프로필 추가가 동작한다.
2. 프로필 전환 시 각 프로필의 저장소 설정이 분리 유지된다.
3. 프로필 삭제 시 남은 프로필로 자동 전환된다.

## 8. 완료 기준

- 하루 1건 이상 로그 저장이 실수 없이 가능
- 사내 저장소 동기화 성공
- 외부 remote로 데이터가 나가지 않음
